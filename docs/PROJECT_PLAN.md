# 프로젝트 실행 계획

## 프로젝트 목표

긴 영어 텍스트를 입력받아, 해당 텍스트만으로 답할 수 있는 퀴즈 질문을 정확히 하나 생성한다.
정답, 선택지, 해설은 출력하지 않는다.

## SQuAD 선택 근거

- SQuAD의 `context`는 영어 Wikipedia를 기반으로 하며 주제가 다양하고 텍스트 품질이 안정적이다.
- 데이터 규모가 충분하여 프로젝트에 적합한 길이의 텍스트를 선별할 수 있다.
- 기존 `question`과 `answers`는 직접 사용하지 않고, `context`만 원천 텍스트로 활용한다.
- 새로운 질문을 생성하고 검수하는 과정을 통해 프로젝트 목적에 맞는 데이터셋을 직접 설계한다.

## 수행 순서

1. SQuAD에서 `context`를 추출하고 중복을 제거한다.
2. 각 context에 대해 새로운 질문 하나를 생성하고 품질을 검수한다.
3. 데이터를 `user → assistant` chat 형식으로 변환한다.
4. Unsloth QLoRA로 실제 파인튜닝을 수행한다.
5. loss 곡선과 학습 로그를 저장하고 모델을 export한다.
6. 학습에 사용하지 않은 텍스트로 베이스 모델과 파인튜닝 모델을 비교한다.
7. 실험 결과를 분석하고 프로젝트 문서를 완성한다.

## 실험 결과 기록 계획

| 항목 | 기록할 내용 |
|---|---|
| 데이터 처리 | context 필터링, 중복 제거, 질문 생성 및 품질 검수 통계 |
| 학습 설정 | 베이스 모델, QLoRA 설정, 하이퍼파라미터와 선택 이유 |
| 학습 과정 | loss, gradient norm, GPU 사용량, 학습 시간 |
| 모델 비교 | 동일 test context 기반 정량 지표와 정성 평가 |
| 모델 배포 | LoRA adapter와 GGUF export 및 추론 방법 |

## 현재 남은 작업

- 새로운 질문 생성 및 품질 검수
- 최종 베이스 모델 선택
- Unsloth 실제 학습
- 비교 평가와 모델 export
