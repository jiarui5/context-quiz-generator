# 项目执行说明（中文）

## 项目目标

输入一段较长的英文文本，只生成一道能够依据文本回答的 Quiz 问题，不输出答案、选项或解释。

## 为什么使用 SQuAD

- SQuAD 的 `context` 来自英文 Wikipedia，主题多样且文本质量稳定。
- 数据量充足，方便筛选长度适中的长文本。
- 我们只把 `context` 当作原始素材，不直接使用已有 `question` 和 `answers`。
- 通过自行生成并检查新问题，可以展示数据集设计过程，这对应评分最高的部分。

## 提交物与制作顺序

1. 从 SQuAD 提取并去重 `context`。
2. 为每个 context 生成一道新问题，并进行质量检查。
3. 转换成 `user → assistant` 训练格式。
4. 使用 Unsloth QLoRA 实际微调。
5. 保存 loss 曲线、日志并导出模型。
6. 使用未训练文本对比基础模型与微调模型。
7. 完成 GitHub README、韩语报告和韩语演示视频。

## 评价标准对应策略

| 老师评分项 | 我们需要展示的证据 |
|---|---|
| 创意与问题定义 20% | 明确目标用户、使用场景和“只生成一道可回答问题”的成功标准 |
| Unsloth 使用 20% | 实际 QLoRA 训练日志、loss 截图、adapter/GGUF export |
| 数据与超参数依据 25% | context 筛选、去重、问题生成、人工质检、参数实验及文献依据 |
| 相比基础模型的改善 15% | 同一测试集的定量指标与人工评分比较 |
| GitHub 质量 10% | 清晰结构、可复现脚本、运行说明和结果截图 |
| 报告与视频 10% | 完整韩语报告及实时对比演示 |
